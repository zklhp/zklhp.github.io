在ARChon环境下运行网易云音乐

May 4, 2015 | 经验分享 | 阅读

首先看一下以前的帖子 这贴不过是整理一下

步骤如下:

  • 下载Google Chrome

我发现Chromium下不如Chrome里稳定 还是用带有谷歌信仰的闭源软件罢

我下载的是64位的免安装的

什么？怎么下免安装的？哈哈 我告诉你我是怎么折腾的

稍微研究一下会发现 谷歌是提供了离线安装包的 但地址是隐藏的 对于42发行版 地址是这个http://dl.google.com/chrome/win/726E532F0545156E/42.0.2311.90_chrome64_installer.exe 很容易发现除了中间一段其他的东东都可以搜出来 于是我们用两个关键字dl.google.com/chrome/win和42.0.2311.90_chrome64_installer.exe搜索 能搜出地址来 这是官方的而且可以解压安装

Update:

现在的地址是

http://dl.google.com/chrome/win/FAC3E7F7D7B29244/42.0.2311.152_chrome64_installer.exe

折腾罢 我就喜欢这么折腾

下面这个没必要

+ 打开NaCl

地址栏输入chrome://flags/#enable-nacl 第一项点启用

如果渲染方面有问题地址栏输入chrome://flags/#ignore-gpu-blacklist(其实就是第一项)点启用

  • 下载ARChon并安装

下载连接在这里

还是选择64位的 这里我们2.1.0版和1.2版都下载 直接下载最新版就好

解压2.1.0版到一个目录 用Chrome的加载正在开发的扩展程序功能安装

  • 下载Crx版的网易云音乐

还是去这里

我们用2.5.4版 地址是这个 因为我发现各版本区别其实不大 关键还是运行环境

拖拽crx文件到扩展程序界面 同意就自动安装了 先别急着运行

  • 修改manifest.json增加目录选择功能

打开Chrome的配置文件夹 在类似Extensions\jalkgaieokmojncbhfgooghjdmplfmpb\1337_0的文件夹里面有manifest.json 修改成类似下面的样子 ...表示省略

像上面那样 要改的是enableExternalDirectory和fileSystem 改了后 还是用Chrome加载这个未打包的扩展 也就是1337_0这个文件夹

然后正常运行就好了

下面的没必要

+ 首次运行网易云音乐

在chrome://apps/里点网易云音乐的图标运行 正常情况下会让你选路径 选一个好记的路径 以后这个路径就是模拟器的内存卡目录了

+ 把ARChon换成1.2版的

还记得我们上面是用的2.1.0版的ARChon么 这里我们把2.1.0版的删掉 换成1.2版的 自然 安装ARChon和crx的步骤要重做一遍

为啥呢 因为我发现1.2版有时候在开始选路径的时候有问题 所以我们用2.1.0版首次运行完成生成模拟内存卡的步骤 再换到1.2版日常使用

以上就是我这几天折腾的成果 目前比较完美的运行着网易云音乐 记录一下 😺
